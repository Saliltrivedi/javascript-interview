Program 23. Prototype Object 

                    Prototype (Definition)

In JavaScript, a prototype is an object from which other objects inherit properties and methods.

Every JavaScript object has a hidden property called [[Prototype]]

This can be accessed using __proto__

Functions have a prototype property that is used when creating objects with new

📌 In short:

A prototype allows objects to share properties and methods.

🔹 Prototypal Inheritance (Definition)

Prototypal inheritance is the mechanism by which one object inherits properties and methods from another object via the prototype chain.

When a property is not found on an object,

JavaScript looks for it in its prototype,

Then the prototype’s prototype, and so on, until null

📌 In short:

Prototypal inheritance means objects inherit directly from other objects.

Program 26. What is the difference between an object created using object literal notation and an object created using a constructor function? 

            An object created using object literal notation is a single instance, while an object 
            created using a constructor function can create multiple instances.
Constructor Function (Old Style)
            
                function User(name, age) {
                    this.name = name;
                    this.age = age;
                }

                const u1 = new User("Salil", 20);
        
            Class (Modern ES6)
                A class is a cleaner, more readable way to do the same thing:

                class User {
                    constructor(name, age) {
                        this.name = name;
                        this.age = age;
                    }
                }

                const u1 = new User("Salil", 20);

Program 31. What is Async and Promises ? 

  
                    A Promise in JavaScript represents a value that will be available now, later, or never (in case of an error).

                    A Promise has 3 states:

                    pending — still working

                    fulfilled — successfully completed

                    rejected — failed
                
                    const myPromise = new Promise((resolve, reject) => {
                    setTimeout(() => {
                        resolve("Data received!");
                    }, 1000);
                    });
                    myPromise.then(result => console.log(result));



                    Program 2

                    const newData= new Promise((resolve,reject)=>{
                            const success=false;
                                
                                if(success){
                                    resolve("Task Completed")
                                }
                                else{
                                    reject("Task in Complete")
                                }
                        })

                        newData.then((result)=>{
                            console.log(result)
                        }).catch((error)=>{
                            console.log(error)
                        })
                
                    What is async and await?

                    async and await are syntactic sugar on top of Promises — they make asynchronous code easier to read and write.

                    A function becomes async when you put the async keyword in front of it.

                    Inside an async function, you can use await to "wait" for a Promise to resolve.

                    Example using async/await:
                
                    async function getData() {
                    const result = await myPromise;   // waits for the promise to finish
                    console.log(result);
                    }

                    getData();
                   // This does the same thing as .then() but looks cleaner.
                
                    Using Promises (.then / .catch)
                    fetch("https://jsonplaceholder.typicode.com/users")
                    .then(response => response.json())
                    .then(data => {
                        console.log("Users:", data);
                    })
                    .catch(error => {
                        console.error("Error:", error);
                    });

                    Explanation:

                    fetch() returns a Promise.

                    We handle the result using .then().

                    Errors are handled with .catch().
                
                    Using async/await (cleaner syntax)
                    async function getUsers() {
                    try {
                        const response = await fetch("https://jsonplaceholder.typicode.com/users");
                        const data = await response.json();
                        console.log("Users:", data);
                    } catch (error) {
                        console.error("Error:", error);
                    }
                    }

                    getUsers();

                    Why this is nicer:

                    Looks like normal synchronous code.

                    Easy to read.

                    Errors handled with try...catch.
                
            | Topic          | Promises                     | async/await                 |
            | -------------- | ---------------------------- | --------------------------- |
            | How it works   | `.then()` and `.catch()`     | `await` and `try...catch`   |
            | Code style     | More callbacks               | Cleaner, readable           |
            | Under the hood | Core async feature           | Built on Promises           |
            | Best for       | Chaining multiple operations | Clean and linear async code |

Program . Arrow Function

An arrow function is a shorter and cleaner way to write functions in JavaScript, introduced in ES6 (2015).

An arrow function is a concise function syntax that does not have its own this, arguments, or prototype.

An arrow function is a shorter function syntax that does not bind its own this. 

 
                    Arrow functions do not have their own this.
                    They use this from the surrounding (lexical) scope.

                    Example
                    const obj = {
                    name: "Salil",
                    greet: function () {
                        setTimeout(() => {
                        console.log(this.name);
                        }, 1000);
                    }
                    };

                    obj.greet(); // Salil


                    ✔ Works because arrow function takes this from greet

                    ❌ Normal function would give undefined

Program 37. What is set

A Set is a JavaScript object used to store unique values of any type. 

                    | Set                 | Array                |
                    | ------------------- | -------------------- |
                    | No duplicate values | Allows duplicates    |
                    | `.has()` is fast    | `.includes()` slower |
                    | No index access     | Indexed              |

               
                Common Set Methods
                const set = new Set();

                set.add(10);
                set.add(20);
                set.add(10); // ignored

                console.log(set.has(20)); // true
                console.log(set.size);    // 2

                set.delete(10);
                set.clear();
               
                const arr = [1, 2, 2, 3, 4];
                const uniqueArr = [...new Set(arr)];

                console.log(uniqueArr); // [1, 2, 3, 4]

Program 43. Event Bubbling Program

Event bubbling in JavaScript is how events travel through the DOM when something happens (like a click). 

               What happens?

                When an event occurs on an element, it:

                Starts at the target element

                Bubbles up to its parent

                Continues up through ancestors (parent → grandparent → document → window)

                So parent elements can react to events that happen on their children.
                

                        

        <div id="parent">
            <button>Click ME</button>
            <button id="child">Click me</button>
        </div>
        


document.getElementById("parent").addEventListener("click", () => {
  console.log("Parent clicked");
});


document.getElementById("child").addEventListener("click", (event) => {
  event.stopPropagation();
  console.log("Child clicked only");
});

Program 47 . Implement your own map() / filter() / reduce(). 

        function myMap(arr, callback) {
        const result = [];
        for (let i = 0; i < arr.length; i++) {
            result.push(callback(arr[i], i, arr));
        }
        return result;
        }

        // Example:
        const nums = [1, 2, 3];
        const doubled = myMap(nums, (x) => x * 2);
        console.log(doubled); // [2, 4, 6]

        

        function myFilter(arr, callback) {
        const result = [];
        for (let i = 0; i < arr.length; i++) {
            if (callback(arr[i], i, arr)) {
            result.push(arr[i]);
            }
        }
        return result;
        }

        // Example:
        const nums = [1, 2, 3, 4];
        const evens = myFilter(nums, (x) => x % 2 === 0);
        console.log(evens); // [2, 4]
