          Undefined/ not defined / null+
        

        

          
            undefined

            A value automatically assigned by JavaScript to a variable that has been declared but not initialized.
            
            Type: undefined

            Example:-

            var a;
            let b;
            const c

            console.log(a,b,c)
            for const c
            So this is not about hoisting or TDZ, it’s a syntax rule: const must have a value at declaration.

            Null

            Represents intentional “no value”.
            Type: object (this is a weird JS quirk)

            Use null when you want to clear a variable or indicate no value intentionally.

            not defined
            Happens when you try to access a variable that hasn’t been declared at all.
            not defined → variable never declared, accessing it throws ReferenceError

            


          

        

      


      

        
var,let and const+

        

          
            var

            => Function-scoped (not block-scoped)

            => Can be redeclared and reassigned

            => Is hoisted and initialized as undefined

            => Old-school JS (mostly avoid now)
            
            let

            => Block-scoped

            => Can be reassigned

            => ❌ Cannot be redeclared in the same scope

            => Hoisted but not initialized (temporal dead zone)

            const 

            Block-scoped

            ❌ Cannot be redeclared

            ❌ Cannot be reassigned

            Must be initialized when declared

            ⚠️ But objects & arrays can still change internally



            yes let and const are hoisted and they are in temporal dead zone for time being.

            let and const stored in seprate memory space

        higher-order function+
        

        

          
              A higher-order function is a function that takes another function as an argument or returns a function.

              const nums = [1, 2, 3];
              const squared = nums.map(n => n * n);

        Call, Apply, Bind +
        

        

          

            call,apply and bind is used for borrowing function

              call()
              Calls the function immediately
              Arguments passed one by one


              apply()

              Calls the function immediately
              Arguments passed as an array

              bind()
              Does NOT call immediately
              Returns a new function with this bound


              | Method | Executes Immediately | Arguments       | Returns         |
              | ------ | -------------------- | --------------- | --------------- |
              | call   | ✅ Yes                | Comma-separated | Function result |
              | apply  | ✅ Yes                | Array           | Function result |
              | bind   | ❌ No                 | Comma / Array   | New function    |


          

        

      


      

        

        Currying in js +
        

        

          
            Currying is a technique where a function with multiple arguments is transformed into a sequence of functions, each taking one argument at a time.

            OR

            Currying is the process of converting a function with multiple arguments into a chain of functions, each taking a single argument.

            Example:-


            function add(a) {
              return function (b) {
                return a + b;
              };
            }

            console.log(add(2)(3))
