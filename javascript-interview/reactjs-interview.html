What is Virtual Dom + 
        The Virtual DOM (VDOM) is a lightweight, in-memory copy of the real DOM (Document Object Model).
        React keeps a virtual representation of the UI in memory.

        When something changes (like data or user input), React updates the Virtual DOM first instead of directly touching the real DOM.

        Then, React figures out the minimal changes needed and updates only those parts in the real DOM.

        This process is called reconciliation.
Why Virtual DOM is Used + 
        Directly updating the real DOM is slow because the browser has to recalculate styles, re-render elements, and repaint the page.

        By using a Virtual DOM:

        React minimizes expensive DOM operations.

        Makes apps faster and smoother.
What is JSX + 
            It is a syntax extension of JavaScript.

            JSX allows you to write HTML-like code inside JavaScript.

            Browsers cannot understand JSX directly, so React transpiles it into regular JavaScript (using tools like Babel) before running it in the browser.

            JSX lets you write HTML inside JavaScript in a way that React can understand and render efficiently. It’s not mandatory, but it makes React code cleaner and easier to read.

import { useState } from 'react';

function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);

  const increment = () => setCount(count + 1);
  const decrement = () => setCount(count - 1);
  const reset = () => setCount(initialValue);

  return { count, increment, decrement, reset };
}

export default useCounter;

    What is a useEffect Cleanup Function? + 
                A cleanup function is a function that React calls before the component unmounts or 
            before the effect runs again (on dependency change). It’s used to:

            Remove event listeners

            Clear timers (setTimeout, setInterval)

            Cancel subscriptions or network requests

            Prevent memory leaks

            Example:

            import { useEffect, useState } from "react";

            function Timer() {
              const [count, setCount] = useState(0);

              useEffect(() => {
                // Side effect: set up an interval
                const intervalId = setInterval(() => {
                  setCount(c => c + 1);
                }, 1000);

                // Cleanup function
                return () => {
                  clearInterval(intervalId);
                  console.log("Cleanup: interval cleared");
                };
              }, []); // empty dependency = runs once on mount

              return 
    Count: {count}
    ;
            }

            export default Timer;


            With dependencies
            useEffect(() => {
              console.log("Effect ran: count =", count);

              return () => {
                console.log("Cleanup before next effect: count =", count);
              };
            }, [count]);


            The cleanup runs before the next effect whenever count changes.

            Then the new effect runs.

    How does React handle priority updates? + 
                React handles priority updates through its Fiber scheduler, which decides what to render first, what can wait, and what can be interrupted. This system is what enables Concurrent React.

            Why priorities exist

          Not all updates are equal:

          Typing into an input → must be immediate

          Filtering a huge list → can wait

          Prefetching data → low priority

          Without priorities, expensive updates would block the UI.

          The core idea

          React assigns a priority to every state update and schedules work accordingly.


          The Scheduler:
          Chooses which Fiber to work on
          Based on priority lanes
          Cooperates with the browser (yields to paint/input)

          | Priority   | Lane type             | Example            |
          | ---------- | --------------------- | ------------------ |
          | Sync       | `SyncLane`            | Click, typing      |
          | Input      | `InputContinuousLane` | Scroll, drag       |
          | Default    | `DefaultLane`         | Data updates       |
          | Transition | `TransitionLane`      | Filtering, sorting |
          | Idle       | `IdleLane`            | Background work    |

          React prioritizes updates using lanes and Fiber scheduling, ensuring urgent updates are fast and non-urgent work doesn’t block the UI.
    How do you prevent unnecessary re-renders?+ 
               Preventing unnecessary re-renders in React is about controlling when components update and keeping props/state stable. Here’s a practical, real-world guide—from most impactful to more situational techniques.


            1. Use React.memo for pure components
            ✅ Prevents re-render when props haven’t changed
            2. Use stable keys in lists
            Unstable keys force React to throw away and recreate components.
            3. Memoize callbacks with useCallback
            Inline functions create new references every render.
            4. Memoize derived data with useMemo
            Avoid recomputing expensive values on every render.
            5. Avoid inline objects and arrays in props
            
            6. Split components by responsibility
            Large components cause widespread re-renders.

            Prevent re-renders by preventing unnecessary prop changes.
